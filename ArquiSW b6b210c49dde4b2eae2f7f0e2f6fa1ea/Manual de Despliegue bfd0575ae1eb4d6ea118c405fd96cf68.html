1. Introducción

Para desplegar la aplicación de ArquiSW se debe tener en cuenta los contenedores que se deben desplegar, esos contenedores lo puedes ver en la Arquitectura e Infraestructura del Sistema, en el diagrama de contenedores, en el cual tienes dos puntos de vista, el del Back del sistema principal y el del Back de ingeniería de requisitos. Independientemente de la vista que elijas observar, los contenedores a desplegar son los siguientes:

  • Base de Datos
  • Storage
  • Back del Sistema Principal
  • Back del Sistema de Ingeniería de Requisitos
  • Front de la Single Page Application

Si observas la arquitectura e infraestructura del sistema, podrás ver que es una aplicación que implementa una arquitectura orientada a servicios, la cual se diferencia de la arquitectura de microservicios porque cada servicio usa la misma base de datos. Es importante tener en cuenta a la hora de desplegar, ya que los servicios dependen de la base de datos, por eso se debe seguir el orden expuesto anteriormente, y el cual puedes seguir en este manual.

2. Contenedor de la base de datos

La base de datos es el contenedor más importante, debido a que va a ser el que almacenará todos los datos que manejará la aplicación, por lo tanto, es el eje principal de su funcionamiento. También, la base de datos, al tratarse de un servicio que guardará información, no existen servicios que lo ofrezcan de manera gratuita. El motor de base de datos utilizado en el proyecto es PostgreSQL.

Sea cual sea el servicio que se elija para desplegar la base de datos, este le generará una URL, con la que posteriormente se configurará en la conexión entre los backend y la base de datos, al contratar el servicio podrán elegir el nombre de la base de datos y la contraseña, datos importantes para la configuración de la conexión.

Y el nombre de usuario y la contraseña de la base de datos, en este caso, se pusieron un usuario y una contraseña de prueba, se recomienda colocar una contraseña más compleja, y el nombre de usuario queda a decisión de la administración.

8. Ramas

En todos los 3 repositorios, se manejan 3 ramas, las cuales son:

  • Develop: Rama donde se realiza el desarrollo, es una rama que pretende usarse para un ambiente no productivo o de pruebas, sino que está hecho para trabajarse en un ambiente local.
  • Quality: Rama espejo de main, que pretende ser la rama intermedia entre develop y main, rama donde llegarán primero los desarrollos de develop, y permitirá probar los desarrollos en un ambiente clon del ambiente real, una vez se tenga todo bien, se pasarían a llevar los cambios a la siguiente rama.
  • Main: Rama principal, que será la rama productiva, en donde se encontrará el ambiente de producción, se recomienda que al desplegar en un ambiente productivo se apunte siempre a esta rama.
